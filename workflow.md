# Signal Workflow

This document outlines the basic workflow for Protocol Signal Lab.

The goal is to collect ecosystem signals, process them with AI, and generate structured research reports.

## Step 1 — Source Monitoring

The system monitors selected sources such as:

- Governance forums
- Protocol GitHub repositories
- Infrastructure project announcements
- Stablecoin ecosystem updates
- Crypto research and news platforms

Sources are listed in `sources.md`.

## Step 2 — Signal Collection

Important updates are collected from the monitored sources, including:

- Governance proposals
- Developer commits and repository activity
- Infrastructure upgrades
- Stablecoin liquidity and ecosystem expansion
- Major protocol announcements

## Step 3 — AI Processing

AI analyzes collected signals to:

- Identify meaningful changes
- Filter noise and repetitive updates
- Extract important insights
- Summarize developments in simple language

## Step 4 — Report Generation

Processed signals are converted into structured research outputs such as:

- Infrastructure Signal Reports
- Ecosystem Development Summaries
- Governance Activity Updates

Reports are stored in the `reports/` directory.

## Step 5 — Continuous Monitoring

The system continuously monitors sources and generates new reports as meaningful signals emerge.

Future versions may automate this workflow using AI agents and automation tools.
